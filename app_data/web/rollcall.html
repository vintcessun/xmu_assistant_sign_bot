                API 使用文档
            

            

                

                    
1. 二维码处理

                    

                        POST /process-qr
                    

                    

                        参数:
                        { "content": "string" }
                    

                    
功能: 处理二维码签到数据

                    

                        参数说明: content 为二维码解析内容，格式为 "/j?p=..."
                        或包含该格式的URL
                    

                    

                        返回值:
                        包含用户签到结果的数组，每个元素包含用户名和签到结果
                    

                    

                        
返回样例:

                        
[
  {
    "user_name": "张三",
    "sign_result": "{\"id\":123,\"status\":\"on_call\"}"
  },
  {
    "user_name": "李四", 
    "sign_result": "{\"error_code\":\"qr_code_expired\",\"message\":\"qr_code_expired\"}"
  }
]

                        

                            数据结构:
                            PushSignData { user_name: String, sign_result:
                                String }
                        

                        

                            说明: sign_result
                            为服务器返回的原始JSON字符串，包含签到ID和状态信息
                        

                    

                

                

                    
2. 签到信息查询

                    

                        POST /process-sign
                    

                    

                        参数:
                        { "user": "string" }
                    

                    
功能: 查询用户的签到状态信息

                    

                        参数说明: user 为QQ号，用于标识用户
                    

                    

                        返回值: 包含用户ID、用户名和详细签到信息的结构化数据
                    

                    

                        
返回样例:

                        
{
  "user_id": "123456",
  "user_name": "张三",
  "sign_data": [
    {
      "type": "number",
      "data": {
        "course_title": "高等数学",
        "course_code": "MATH101",
        "activity_id": 1001,
        "instructors": ["王教授", "李教授"],
        "sign_num": 45,
        "student_num": 60,
        "number_code": "123456",
        "status": "absent"
      }
    },
    {
      "type": "radar",
      "data": {
        "course_title": "大学物理",
        "course_code": "PHYS201",
        "activity_id": 1002,
        "instructors": ["赵教授"],
        "sign_num": 30,
        "student_num": 50,
        "status": "absent"
      }
    },
    {
      "type": "qr",
      "data":{
        "course_title": "大学物理",
        "course_code": "PHYS201",
        "activity_id": 1002,
        "instructors": ["赵教授"],
        "sign_num": 30,
        "student_num": 50,
        "status": "on_call_fine"
      }
    }
  ]
}

                        

                            数据结构:
                            SignResponse { user_id: String, user_name:
                                String, sign_data: Vec<SignData> }
                        

                        

                            SignData 类型: number(数字签到), radar(雷达签到),
                            qr(二维码签到)
                        

                        

                            状态说明: "absent"=未签到, "on_call_fine"=已签到
                        

                    

                

                

                    
3. 自动签到

                    

                        POST /process-auto-sign
                    

                    

                        参数:
                        { "user": "string" }
                    

                    
功能: 执行自动签到操作

                    

                        参数说明: user 为QQ号，用于标识用户
                    

                    

                        返回值:
                        自动签到结果数组，包含雷达签到、数字签到、二维码签到的成功/已签到状态
                    

                    

                        
返回样例:

                        
[
  {
    "type": "radar",
    "data": {
      "status": "success",
      "data": {
        "course_name": "高等数学",
        "student_location": "教学楼A区",
        "latitude": 24.123456,
        "longitude": 118.123456,
        "student_distance": 15.5
      }
    }
  },
  {
    "type": "number",
    "data": {
      "status": "already_signed",
      "data": {
        "course_name": "大学英语"
      }
    }
  },
  {
    "type": "qr",
    "data": {
      "status": "success",//这里success无法签到二维码
      "data": {
        "course_name": "计算机基础"
      }
    }
  }
]

                        

                            数据结构: Vec<AutoSignData>，其中
                            AutoSignData 可以是:
                        

                        
    
                            
  • radar: 雷达签到 (success/already_signed)
    • 
                            
  • number: 数字签到 (success/already_signed)
    • 
                            
  • qr: 二维码签到 (success/already_signed)
    • 
                        

                        

                            success 状态包含详细签到信息，already_signed
                            状态仅包含课程名称
                        

                    

                

                

                    
4. 特定自动签到

                    

                        POST /process-spec-sign
                    

                    

                        参数:
                        { "user": "string", "rollcall_id": "string" }
                    

                    
功能: 执行自动签到操作

                    

                        参数说明: user 为QQ号，用于标识用户；rollcall_id
                        为签到ID，用于指定签到活动
                    

                    

                        返回值:
                        自动签到结果对象，包含雷达签到、数字签到、二维码签到的成功/已签到状态
                    

                    

                        
返回样例:

                        
{
  "type": "radar",
  "data": {
    "status": "success",
    "data": {
      "course_name": "高等数学",
      "student_location": "教学楼A区",
      "latitude": 24.123456,
      "longitude": 118.123456,
      "student_distance": 15.5
    }
  }
}


                        
数据结构: AutoSignData 可以是:

                        
    
                            
  • radar: 雷达签到 (success/already_signed)
    • 
                            
  • number: 数字签到 (success/already_signed)
    • 
                            
  • qr: 二维码签到 (success/already_signed)
    • 
                        

                        

                            success 状态包含详细签到信息，already_signed
                            状态仅包含课程名称
                        

                    

                

                

                    
5. 推送签到

                    

                        POST /process-push-sign
                    

                    

                        参数:
                        { "rollcall_id": "string" }
                    

                    

                        功能: 推送签到给指定签到ID的所有用户
                    

                    

                        参数说明: rollcall_id
                        为签到ID，可通过签到信息查询接口获得
                    

                    

                        返回值:
                        包含用户签到结果的数组，每个元素包含用户名和签到结果
                    

                    

                        
返回样例:

                        
[
  {
    "user_name": "张三",
    "sign_result": "{\"id\":1001,\"status\":\"success\",\"distance\":12.5}"
  },
  {
    "user_name": "李四",
    "sign_result": "{\"id\":1002,\"status\":\"success\",\"distance\":8.3}"
  }
]

                        

                            数据结构:
                            Vec<PushSignData>，与二维码处理接口相同
                        

                        

                            说明: 对于雷达签到，sign_result
                            包含距离信息；对于数字签到，包含签到结果状态
                        

                        

                            支持类型: 数字签到、雷达签到（自动选择最近位置）
                        

                    

                

                

                    
底层函数说明

                    

                        所有功能基于 rust 实现，提供高性能的异步签到处理：
                    

                    
    
                        
  • 
                            broadcast_qr_sign_data:
                            广播二维码签到，支持多用户并发处理
                        
    • 
                        
  • 
                            sign_core_data:
                            获取用户签到状态，包含详细课程信息
                        
    • 
                        
  • 
                            auto_sign_core_data:
                            自动签到，支持雷达、数字、二维码签到类型
                        
    • 
                        
  • 
                            push_sign_core_data:
                            推送签到，针对特定签到ID批量处理
                        
    • 
                    

                    

                        
使用示例:

                        
# 二维码处理
curl -X POST http://localhost:8083/process-qr   -H "Content-Type: application/json"   -d '{"content": "/j?p=0~123!1~456!2~qr!3~data123"}'

# 签到信息查询  
curl -X POST http://localhost:8083/process-sign   -H "Content-Type: application/json"   -d '{"user": "123456"}'

# 自动签到
curl -X POST http://localhost:8083/process-auto-sign   -H "Content-Type: application/json"   -d '{"user": "123456"}'

# 推送签到
curl -X POST http://localhost:8083/process-push-sign   -H "Content-Type: application/json"   -d '{"rollcall_id": "1001"}'
